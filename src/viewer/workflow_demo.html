このページでは、図面を受け取ってから概算見積を画面表示するまでの内部フローをまとめています。 図面の読取、形状の把握、工程の推定、マスター引き当て、金額組み立てを段階ごとに分けて示します。

全体像

図面入力から見積表示までの6ステップ

workflow
01

図面を受け取る

PDF や画像を入力として受け取り、ページ全体の表示用画像を準備します。

  • 原図面のレンダリング
  • 表示ビューの切り出し準備
  • 案件キーの付与
02

OCR的に読取する

寸法、材質、表面処理、穴注記、断面記号などを読取り候補として整理します。

  • 図面タイトル欄の読取
  • 注記と寸法の抽出
  • 不足時は補足指定を併用
03

形状と工程を推定する

外形、穴、タップ、曲げ、座ぐりなどから、対象部品の加工プロセスを組み立てます。

  • 板金 / 切削 / 長尺材の分類
  • 穴数や曲げ数のカウント
  • 工程候補のルート化
04

マスターを引き当てる

材質、工程、表面処理、熱処理をデモ用マスターに照合し、単価と密度を取得します。

  • 材質 family と密度
  • 工程ごとの段取り費 / 実行単価
  • 処理ごとの基本費 / 個数単価
05

概算金額を組み立てる

素材費、工程費、表面処理、熱処理を合算して、見積金額と前提を作ります。

  • 素材寸法から重量換算
  • 工程 count による加算
  • 指示なし項目は `なし` 扱い
06

比較画面として表示する

元図面の切り出し、簡易 2D CAD、概算金額、注意書きを1ページにまとめます。

  • 比較ビューの並列表示
  • 見積内訳の表示
  • 根拠と注意の明示

データの流れ

どの情報を使って、どの出力を作っているか

入力

Input

PDF / JPG / PNG の図面本体と、必要に応じた補足指定を受け取ります。

  • 図面ファイル
  • 材質や処理の追加入力
  • 案件名、品名

読取結果

Extraction

寸法、穴仕様、曲げ、材料、表面処理の候補をテキスト化して保持します。

  • 外形寸法
  • 穴径 / タップ / 座ぐり
  • 表面処理 / 熱処理

工程推定

Routing

形状と注記から加工順序を決め、見積に使う工程 count へ落とします。

  • ブランク
  • 曲げ / 穴加工 / タップ
  • 切削の外形 / ポケット / 仕上げ

マスター照合

Master Lookup

内部のデモ用マスターから、材質密度と単価テーブルを引き当てます。

  • 材質マスター
  • 工程マスター
  • 表面処理 / 熱処理マスター

金額計算

Pricing

重量と工程数を使って素材費と加工費を積み上げ、概算合計を算出します。

  • 素材費
  • 工程費
  • 処理費

画面出力

Viewer

比較ビュー、見積結果、引き当て根拠、注意書きをブラウザで表示します。

  • 比較用 HTML
  • 簡易 2D CAD SVG
  • estimate_input.json

このデモで見せているポイント

単なる金額表示ではなく、見積の根拠も併せて見せる構成です

形状の理解

  • 図面全体ではなく、見せたいビューを切り出して比較
  • 簡易 2D CAD 側で、穴や曲げの構成を見やすく整理
  • どの形状が見積に効いているかを説明しやすい

見積根拠の明示

  • 材質、工程、表面処理、熱処理の引き当て先を表示
  • 図面記載か推定か、ユーザー指定かを区別
  • 不明点は `なし` や仮設定として注意書きに残す

内部データの形

  • `manifest.json` で比較ビュー構成を定義
  • `estimate_input.json` で見積前提を定義
  • 共通 JS で金額計算と表示を共通化

推定が入る箇所

  • OCR 読取が曖昧な場合は図面の見た目から補完
  • 工程分類は現状デモ向けの簡略ロジック
  • 加工可否、公差、工数最適化、数量割引は未反映

ページに出るもの

各デモページで実際に見せている要素

図面比較ビュー

比較用

元図面の切り出し画像と簡易 2D CAD を並べ、形状の対応を確認できます。

概算見積

金額

素材費、工程費、表面処理、熱処理を分けて表示し、合計金額を確認できます。

引き当て根拠

根拠

材質や処理が図面記載なのか、推定なのか、補足指定なのかを日本語で表示します。

注意書き

前提

熱処理なし扱い、不明項目の仮設定、デモ単価であることなどを明示します。
