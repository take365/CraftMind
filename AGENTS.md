# Codex ワークフローエージェント

このリポジトリは、図面から工程見積りと可視化結果を段階的に作るための作業台です。
現行の主系は `2D図面比較 + 自動概算見積` のデモ生成です。
通常の新規作業では 2D デモ系を優先します。

## 目的

入力図面から次を作る。

1. 図面読取結果
2. 工程ルート案
3. 材質 / 工程 / 表面処理 / 熱処理の引き当て
4. 概算金額
5. 図面比較ビュー付きの HTML デモ
6. 必要に応じて工程メモや検証メモ

## 基本方針

- 図面入力は「一発回答」ではなく、検証可能な作業項目として扱う
- 推測は必ず前提として明示する
- 完璧な幾何形状より、工程の説明と検証を優先する
- 不明点がある段階では、断定せず、確からしい仮定を記録して次へ進む
- 元図面や取得した参照ファイルは上書きしない
- 再現性を優先し、同じ図面を再投入したときに同じ命名規約、同じ更新対象、同じチェックを通す
- 見積デモを追加した場合は、HTML だけでなく入力 JSON、生成スクリプト、一覧導線も更新する

## 現行の正式フロー

新しい図面を追加するときは、原則として以下の 2D 自動見積フローを使う。

1. 図面から比較ビュー候補を切り出す
2. 簡易 2D CAD SVG を作る
3. OCR的に図面を読み、材質、工程、表面処理、熱処理の候補を整理する
4. `estimate_input.json` を作り、見積前提を固定する
5. デモ用マスターを使って概算見積を表示する HTML を用意する
6. `index.html` と README を更新し、デモ一覧から遷移できるようにする

## CAD入力の扱い

PDF / 画像だけでなく、`DXF` と `STEP` の入力も扱ってよい。
ただし、現時点の正式運用では CAD からの 3D プレビュー生成は対象外とする。

- `DXF`
  - 2D 形状要素の抽出対象
  - 外形、線分、円、穴候補、概略寸法を拾う
- `STEP`
  - 3D 形状要素の抽出対象
  - bbox、面数、円筒面、穴候補、概略寸法を拾う
- `STEP` の実メッシュ表示や 3D viewer 化は行わない
- `STEP` から見せるのは、抽出メトリクス、簡易投影、feature 要約までに留める
- 見積表示は既存の 2D 自動見積ページ構成に寄せる

## 作業フロー

### 1. 読取

図面の事実をまず抜き出す。ここでは解釈しすぎない。

- 部品種類を分類する
- 材質、板厚、外形寸法、穴、曲げ、注記を抜く
- 読めた値と、読めなかった値を分けて記録する
- 必要なら「座標表」を別に作る

#### 読取の粒度

- 第1段階: 仕様の事実抽出
- 第2段階: 寸法の座標化
- 第3段階: 3D化に必要な解釈の固定

### 2. 工程計画

工程ルートを作る。

- 各工程を設備種別に対応づける
- 必要工具、能力不足、曖昧な指示を洗い出す
- 加工順が妥当か確認する

### 2.5. 見積引き当て

工程案と図面記載をもとに、見積表示に必要な引き当て情報を固定する。

- 材質を決める
- 表面処理を決める
- 熱処理を決める
- 各工程の count を決める
- 不明なものは `なし` または仮定として明示する

### 3. 妥当性確認

工程案が図面と設備の両方に対して無理がないかを見る。

- 現有設備で現実的か確認する
- 図面の制約に合っているか確認する
- 不一致があれば、3D化の前に工程を修正する

### 4. 可視化作成

現行の主系は 2D 比較ビューである。必要な場合のみ 3D を追加する。

- 原図面の注釈付き画像を出す
- 比較用の crop 画像を出す
- 簡易 2D CAD SVG を出す
- HTML で比較ビューと見積をまとめる
- 3D を作る場合は補助成果物として扱う

CAD 入力デモでは次を優先する。

- 入力 CAD の簡易レンダリング
- 見積用 feature スケッチ
- 抽出メトリクスの表示
- 概算見積と注意書き

次は行わない。

- `STEP` の 3D viewer
- `STEP` のメッシュ変換
- Web 向けの実形状 3D プレビュー

### 5. レビューと梱包

最終成果物を使える形でまとめる。

- ステージ一覧と前提条件を出す
- HTML などの viewer 資産を用意する
- 図面読取から最終成果物までの追跡を残す

## 出力規約

作業用成果物は `src/workflow/` に置く。可視化資産は `src/viewer/` に置く。
生成スクリプトは `src/compare/` に置く。
見積マスターは `src/master/demo/` を参照する。

推奨ファイル:

- `src/workflow/intake.md`
- `src/workflow/dimension_table.md`
- `src/workflow/process_plan.md`
- `src/workflow/validation.md`
- `src/workflow/final_summary.md`

2D 自動見積デモの必須ファイル:

- `src/compare/<part_key>_2d.py`
- `src/viewer/demos/<part_key>_2d.html`
- `src/viewer/assets/<part_key>_2d/manifest.json`
- `src/viewer/assets/<part_key>_2d/estimate_input.json`
- `src/viewer/assets/<part_key>_2d/*.png`
- `src/viewer/assets/<part_key>_2d/*.svg`

`part_key` は図面を一意に区別できる短い英数字スネークケースにする。

## 見積入力の固定ルール

`estimate_input.json` には少なくとも次を持たせる。

- `part_key`
- `part_name`
- `drawing_reference`
- `service_family`
- `stock_dimensions_mm`
- `material`
- `surface_treatment`
- `heat_treatment`
- `process_plan`
- `assumptions`
- `warnings`

各引き当てには状態を残す。

- `drawing_spec`
- `user_specified`
- `estimated`
- `fallback_none`
- `assumed`

既存デモとの互換のため、次の旧状態値も許容する。

- `explicit`
- `inferred`
- `fallback_default`

状態が `estimated` または `assumed` の場合は、理由を必ず残す。
状態ごとの理由は、各項目の `note`、または `assumptions` / `warnings` に残す。

## 雛形生成

新しい 2D 自動見積デモを始めるときは、まず雛形生成スクリプトを使う。

```powershell
python src/compare/scaffold_2d_demo.py sample_part ^
  --product-name "サンプル部品" ^
  --drawing-reference "meviy_quotation_examples/pdfs/REPLACE_ME.pdf"
```

このスクリプトは、比較スクリプト、デモ HTML、`estimate_input.json`、`manifest.json`、
workflow テンプレート、および `index.html` と README の導線を初期化する。

## 一覧更新ルール

新しいデモを追加したら、次も更新する。

- `src/viewer/index.html`
- `src/README.md`
- `src/viewer/README.md`

`index.html` では最低限次を追加する。

- デモ名
- 2D比較 + 自動概算見積 の説明
- 対象部品の要約
- 概算金額
- デモ HTML へのリンク

## 完了条件

新規デモ作成が完了したと判断できる条件は次の通り。

1. `src/compare/<part_key>_2d.py` でアセットを再生成できる
2. `src/viewer/demos/<part_key>_2d.html` をブラウザで開ける
3. 元図面の切り出し画像と簡易 2D CAD が両方表示される
4. 見積結果、引き当て根拠、注意書きが表示される
5. `index.html` から遷移できる
6. README の URL 一覧に載っている

## 更新時チェックリスト

作業後は最低限次を確認する。

- asset パスが `demos/` 配下から見て壊れていないか
- `estimate_input.json` の材質、工程、表面処理、熱処理に状態が入っているか
- 図面記載とユーザー補足指定が競合する場合、どちらを優先したか注意に書いたか
- HTML の文言が日本語で読めるか
- 見積結果が空欄や `NaN` になっていないか

## 読取精度の扱い

- 位置関係を正確にしたい場合は、まず寸法表を作る
- その後に3Dモデルを作る
- 3D化の前に、基準面・基準線・穴中心座標を明示する
- 曖昧な注記は「確定値」と分けて「仮定」として残す

### 座標化の出力要件

- 外形は、寸法値だけでなく頂点列で記録する
- 穴・タップ・切欠きは、必ず feature ID を振る
- 各 feature には少なくとも次を持たせる
  - 種別
  - 加工工程
  - 基準
  - 中心座標または頂点座標
  - 径、ねじ、公差、確度
- 連鎖寸法から導いた値は、導出式も残す
- 「図面で読めた値」と「対称条件から導いた値」を分けて書く
- Stage 1 で加工する要素は、外形と穴を別表にする
- 3Dモデルは、寸法表に書いた座標をそのまま使う

### 推奨する座標ファイルの粒度

- `intake.md`: 図面の事実抽出
- `dimension_table.md`: 基準、外形頂点、feature 座標
- `stage_01_model.md`: 第1工程で実際に加工する輪郭と穴
- 必要なら `stage_02_model.md` 以降で二次加工位置を分離する

## 3D の扱い

3D は補助用途とし、通常の新規追加では必須にしない。

- 2D 自動見積が主成果物
- 3D を追加する場合でも、現行の 2D デモ構成と競合しない配置にする
- 3D のみを追加して 2D デモと一覧更新を省略しない
- `STEP` 入力対応を行っても、原則として 3D 表示までは作らない
- `STEP` では 2D / 指標ベースの要約表示を正式運用とする

## エスカレーション

図面のルートを十分な確度で決められない場合は、無理にモデル化しない。

- 妥当性確認の段階で止める
- 失敗理由を明記する
- 必要なら追加確認事項を列挙する
